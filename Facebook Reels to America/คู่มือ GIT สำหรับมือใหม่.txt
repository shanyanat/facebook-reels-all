------------------------------------------------------------
 1. GIT คืออะไร (แบบเข้าใจง่าย)
------------------------------------------------------------

Git คือ "ระบบเซฟอัจฉริยะ" สำหรับโฟลเดอร์ไฟล์ของคุณ

ลองนึกถึงปุ่ม SAVE ในเกม แต่เก่งกว่ามาก:
  - มันจำการเซฟ "ทุกครั้ง" (เรียกว่า "commit") ไม่ใช่แค่ครั้งล่าสุด
  - คุณย้อนกลับไปการเซฟเก่า ๆ ได้ ถ้ามีอะไรพัง
  - มันคัดลอกไฟล์ไปเครื่องอื่นได้อย่างปลอดภัย

ประโยชน์ที่สำคัญที่สุดสำหรับคุณ:
  Git รู้ว่าไฟล์ไหนควรคัดลอก และไฟล์ไหน "ห้ามคัดลอกเด็ดขาด"
  นี่แก้ปัญหาที่เคยทำให้ไฟล์ contents.json พังมาแล้ว
  (สาเหตุคือคัดลอกไฟล์ข้อมูลข้ามเครื่องด้วยมือ)


------------------------------------------------------------
 2. หัวใจสำคัญ: แยก CODE กับ DATA
------------------------------------------------------------

โปรเจกต์ของคุณมีไฟล์ 2 ประเภทที่ต่างกันมาก:

  CODE (โค้ด) = ตัวเครื่องมือ (เหมือนกันทุกเครื่อง)
       เช่น: bot.py, parse_analysis.py, phases/, extension/
       --> คัดลอกไปทุกเครื่องได้อย่างปลอดภัย

  DATA (ข้อมูล) = ไฟล์งานของแต่ละเครื่อง (ต่างกันทุกเครื่อง)
       เช่น: data/, pages/, complete/, โปรไฟล์ Chrome
       --> ห้ามคัดลอกเด็ดขาด แต่ละเครื่องสร้างของตัวเอง

Git ถูกตั้งค่าให้เก็บเฉพาะ CODE เท่านั้น
ส่วน DATA จะถูกละเว้นทั้งหมด จึงไม่มีทางถูกคัดลอกผิดพลาดอีก


------------------------------------------------------------
 3. สิ่งที่ตั้งค่าไว้ให้แล้ว (เมื่อ 7 มิ.ย. 2026)
------------------------------------------------------------

ในโฟลเดอร์:
  C:\Claude code\ปรึกษาส่วนตัว\Facebook Reels All\Facebook Reels to America

ได้ทำขั้นตอนเหล่านี้ไว้:

  (a) สร้างไฟล์ชื่อ ".gitignore"
      ไฟล์นี้คือ "รายการห้ามคัดลอก" บอกให้ Git ละเว้น:
        data/          (contents.json และไฟล์สำรอง)
        pages/         (บรีฟ, ไฟล์งาน, คลิป - 563 MB)
        complete/      (รีลที่เสร็จแล้ว - 6.8 GB)
        chrome-bot/    (ล็อกอินของ Chrome)
        __pycache__/   (แคชของ Python ที่สร้างอัตโนมัติ)
        *.bak, *.broken, *.lock, *.corrupt-*  (ไฟล์เซฟตี้)
        .claude/settings.local.json  (ตั้งค่าเฉพาะเครื่อง)

  (b) เริ่มใช้ Git ในโฟลเดอร์ ("git init")

  (c) บันทึกสแนปช็อตแรก ("commit")
      เซฟไฟล์ CODE ไว้ 24 ไฟล์ โดยไม่มีโฟลเดอร์ข้อมูลก้อนใหญ่
      รวมอยู่เลย - ตรงตามที่ต้องการ

  (d) ตั้งค่าให้ชื่อไฟล์ภาษาไทยแสดงผลถูกต้อง

ผลลัพธ์: เครื่องนี้มีประวัติการเซฟครบถ้วน และมีตาข่ายนิรภัย
ให้โค้ดของคุณแล้ว ส่วนโฟลเดอร์ข้อมูลก็ปลอดภัย


------------------------------------------------------------
 4. วิธีใช้ GIT ทุกวัน (มีแค่ 3 คำสั่งที่ต้องใช้)
------------------------------------------------------------

ทุกครั้งที่แก้โค้ดแล้วอยากเซฟสแนปช็อต:

  ขั้นที่ 1 - เปิด Warp Terminal ในโฟลเดอร์โปรเจกต์:
     cd "C:\Claude code\ปรึกษาส่วนตัว\Facebook Reels All\Facebook Reels to America"

  ขั้นที่ 2 - เตรียมการเปลี่ยนแปลง (แปลว่า "เตรียมทุกอย่างเพื่อเซฟ"):
     git add -A

  ขั้นที่ 3 - เซฟสแนปช็อต พร้อมโน้ตสั้น ๆ ว่าแก้อะไรไป:
     git commit -m "แก้บั๊กตอนดาวน์โหลดวิดีโอ" 

แค่นี้เอง คุณเพิ่งสร้างเซฟถาวรที่ย้อนกลับมาได้เสมอแล้ว

ดูประวัติการเซฟเมื่อไหร่ก็ได้:
     git log --oneline

(กดตัวอักษร  q  เพื่อออกจากหน้าประวัติ ถ้ามันเต็มจอ)


------------------------------------------------------------
 5. วิธีย้อนกลับ ถ้ามีอะไรพัง
------------------------------------------------------------

ถ้าแก้ไฟล์ไปแล้วอยากทิ้งการแก้นั้น (ก่อนที่จะ commit)
แล้วกลับไปเวอร์ชันที่เซฟล่าสุด:

     git restore <ชื่อไฟล์>

  ตัวอย่าง:
     git restore bot.py

ดูรหัสของเซฟเก่า:
     git log --oneline
  (เซฟแต่ละครั้งมีรหัสสั้น ๆ เช่น  f4f5c9a )

ถ้าต้องการย้อนกลับไปเซฟเก่าจริง ๆ อย่าเดาเอง
ให้เปิด Claude Code แล้วถาม เพราะขั้นตอนนี้อาจลบงานล่าสุดได้
ถ้าทำผิด จึงปลอดภัยกว่าถ้าขอความช่วยเหลือ


------------------------------------------------------------
 6. การซิงก์ไปเครื่องอื่น (ขั้นต่อไป - ยังไม่ได้ทำ)
------------------------------------------------------------

ตอนนี้ Git ปกป้องโค้ดของคุณเฉพาะ "เครื่องนี้" เท่านั้น

ถ้าจะคัดลอกโค้ดไปเครื่องสำรองอื่นอย่างปลอดภัย คุณต้องมี
"สำเนาออนไลน์" เรียกว่า "remote" ตัวเลือกฟรีมาตรฐานคือ
repository แบบ "ส่วนตัว" บน GitHub (มีแค่คุณคนเดียวที่เห็น)

เมื่อเชื่อมต่อแล้ว ขั้นตอนจะกลายเป็น:

  ที่เครื่องนี้ (หลัง commit):
     git push          <- อัปโหลดโค้ดล่าสุดขึ้นออนไลน์

  ที่เครื่องอื่น:
     git pull          <- ดาวน์โหลดโค้ดล่าสุดอย่างปลอดภัย
                          (จะไม่ดึง data/ หรือ pages/ มาด้วยเด็ดขาด
                           เพราะมี .gitignore กันไว้)

นี่คือข้อดีก้อนใหญ่: การอัปเดตทุกเครื่องเหลือแค่คำสั่งเดียว
ที่ปลอดภัย แทนการคัดลอกไฟล์ด้วยมือ (ซึ่งเป็นต้นเหตุที่ทำให้
ข้อมูลพังในอดีต)

ขั้นตอนนี้ยังไม่ได้ตั้งค่า เพราะมันส่งโค้ดของคุณขึ้นบริการ
ออนไลน์ เมื่อพร้อมแล้วให้เปิด Claude Code แล้วพิมพ์ว่า:
"ช่วยเชื่อมต่อ GitHub แบบส่วนตัวให้หน่อย" แล้วผมจะแนะนำทีละขั้น


------------------------------------------------------------
 7. กฎทอง (จำให้ขึ้นใจ)
------------------------------------------------------------

  1. ห้ามคัดลอก data/, pages/, complete/ ข้ามเครื่องด้วยมือ
     เด็ดขาด ปล่อยให้แต่ละเครื่องสร้างของตัวเอง (ใช้ git pull
     หลังเชื่อม GitHub แล้ว มันจะคัดลอกแค่โค้ด)

  2. commit บ่อย ๆ เซฟเล็ก ๆ พร้อมโน้ตที่ชัดเจนดีที่สุด

  3. ถ้า contents.json ดูเหมือนจะพัง ให้รัน:
        py repair_contents.py
     (อันนี้แยกจาก Git - มันใช้ซ่อมไฟล์ข้อมูล)

  4. ถ้าไม่แน่ใจ ให้ถาม Claude Code ก่อนรันคำสั่งใด ๆ
     ที่ลบหรือเขียนทับประวัติ


------------------------------------------------------------
 8. ตารางสรุปคำสั่งแบบเร็ว
------------------------------------------------------------

  git add -A                 เตรียมการเปลี่ยนแปลงทั้งหมดเพื่อเซฟ
  git commit -m "โน้ต"        เซฟสแนปช็อตพร้อมโน้ต
  git status                 ดูว่ามีอะไรเปลี่ยน / เตรียมเซฟอะไรไว้
  git log --oneline          ดูประวัติการเซฟ
  git restore <ไฟล์>          ยกเลิกการแก้ที่ยังไม่ได้เซฟ
  git push                   อัปโหลดขึ้น GitHub (หลังตั้งค่า)
  git pull                   ดาวน์โหลดจาก GitHub (หลังตั้งค่า)
