================================================================================
  คู่มือการใช้งานระบบ Facebook Reels Automation
  (อ่านจากบนลงล่างตามลำดับขั้นตอน)
================================================================================

--------------------------------------------------------------------------------
ภาพรวมระบบทำงานอย่างไร
--------------------------------------------------------------------------------

ระบบนี้ช่วยสร้างคลิปวิดีโอสำหรับ Facebook Reels โดยอัตโนมัติ ประกอบด้วย 4 ขั้นตอนหลัก:

  1. เตรียมไฟล์ Brief (.txt) และวาง Character Sheet ลงในโฟลเดอร์ของเพจ
  2. รัน images  → บอทเปิด ChatGPT สร้างภาพ Storyboard + ภาพซีนทั้งหมด + ภาพ Thumbnail
  3. รัน videos  → บอทเปิด Google Flow สร้างวิดีโอจากภาพแต่ละซีน
  4. รัน archive → ย้ายไฟล์ทั้งหมดไปยังโฟลเดอร์ ready/ เพื่อนำไปตัดต่อ

ทุกคำสั่งพิมพ์ใน Terminal (Warp) และต้องอยู่ที่โฟลเดอร์โปรเจกต์เสมอ:
  cd "C:\Claude code\ปรึกษาส่วนตัว\Facebook Reels to America"


--------------------------------------------------------------------------------
โครงสร้างโฟลเดอร์ (ทำความเข้าใจก่อนใช้งาน)
--------------------------------------------------------------------------------

Facebook Reels to America/
│
├── pages/                        ← โฟลเดอร์หลักของทุกเพจ
│   └── page-ชื่อเพจ/
│       ├── briefs/               ← วาง Character Sheet และไฟล์ Brief (.txt) ที่นี่
│       ├── working/              ← ระบบสร้างไฟล์ระหว่างทำงานที่นี่ (อย่าแตะ)
│       └── ready/                ← ไฟล์สำเร็จพร้อมตัดต่อ CapCut อยู่ที่นี่
│           └── reel_XXXX/
│               ├── ชื่อbrief.txt
│               ├── storyboard.png
│               ├── thumbnail.png     ← ภาพ Thumbnail (ถ้า Brief มี THUMBNAIL PROMPT)
│               ├── scene-01.mp4
│               └── scene-10.mp4
│
├── data/
│   └── contents.json             ← ฐานข้อมูลโปรเจกต์ทั้งหมด (อย่าแก้ไขเอง)
│
├── bot.py                        ← ไฟล์คำสั่งหลัก
└── monitor.py                    ← โปรแกรมเฝ้าดูไฟล์ (เปิดทิ้งไว้ถ้าต้องการ)


--------------------------------------------------------------------------------
รายการคำสั่งทั้งหมด (สรุปสั้น ๆ)
--------------------------------------------------------------------------------

  py bot.py addpage <ชื่อเพจ>                    สร้างโฟลเดอร์เพจใหม่
  py bot.py status                               ดูสถานะโปรเจกต์ทั้งหมด
  py bot.py queue                                นำ Brief ใหม่เข้าคิวในระบบ
  py bot.py images reel_XXXX                     สร้างภาพ (Storyboard + ภาพซีนทั้งหมด)
  py bot.py videos reel_XXXX                     สร้างวิดีโอจากภาพซีน
  py bot.py archive reel_XXXX                    ย้ายไฟล์สำเร็จแล้วไปยัง ready/
  py bot.py collect                              รวบรวมโฟลเดอร์ ready/ ทุกเพจไปไว้ใน complete/ (ใช้หลัง archive)
  py bot.py updateprompts reel_XXXX              ดึง Prompt ใหม่จากไฟล์ .txt (ไม่รีเซ็ตความคืบหน้า)
  py bot.py renamepage <ชื่อเก่า> <ชื่อใหม่>    เปลี่ยนชื่อโฟลเดอร์เพจ (ทำทุกอย่างครบในขั้นตอนเดียว)


================================================================================
  ขั้นตอนการใช้งาน (ทีละขั้นตามลำดับ)

────────────────────────────────────────────────────────────────────────────────
ขั้นตอนที่ 0 — เปิด Terminal และไปที่โฟลเดอร์โปรเจกต์
────────────────────────────────────────────────────────────────────────────────

ทุกครั้งที่จะใช้งาน ต้องเปิด Terminal (Warp) แล้วพิมพ์:

  cd "C:\Claude code\ปรึกษาส่วนตัว\Facebook Reels to America"

แล้วกด Enter ก่อนเสมอ ถ้าไม่ทำขั้นตอนนี้ คำสั่งจะทำงานไม่ได้


────────────────────────────────────────────────────────────────────────────────
ขั้นตอนที่ 1 — สร้างโฟลเดอร์เพจใหม่ (ทำครั้งเดียวต่อเพจ)
────────────────────────────────────────────────────────────────────────────────

ใช้เมื่อ: ต้องการเพิ่มเพจ Facebook ใหม่เข้าระบบ (ทำเพียงครั้งเดียวต่อเพจ)

คำสั่ง:
  py bot.py addpage page-ชื่อเพจ

ตัวอย่าง:
  py bot.py addpage page-Noble-Handiwork
  py bot.py addpage page-babyshark

ผลลัพธ์: ระบบสร้างโฟลเดอร์ pages/page-ชื่อเพจ/briefs/ และ pages/page-ชื่อเพจ/ready/ ให้อัตโนมัติ

สิ่งที่ต้องทำหลังรันคำสั่งนี้:
  → วาง Character Sheet (ภาพตัวละคร .png หรือ .jpg) ลงใน pages/page-ชื่อเพจ/briefs/
  → Character Sheet จะอยู่ที่นั่นถาวร ใช้ซ้ำได้ทุกโปรเจกต์ของเพจนั้น

หมายเหตุ: ชื่อเพจห้ามมีช่องว่าง ให้ใช้ขีดกลาง (-) แทน
  ✓ page-Noble-Handiwork
  ✗ page Noble Handiwork


────────────────────────────────────────────────────────────────────────────────
ขั้นตอนที่ 2 — วางไฟล์ Brief และนำเข้าคิว
────────────────────────────────────────────────────────────────────────────────

ใช้เมื่อ: มีไฟล์ Brief (.txt) ใหม่ที่ต้องการให้ระบบสร้างคลิป

2.1 วางไฟล์ Brief
  → คัดลอกไฟล์ .txt ลงในโฟลเดอร์ pages/page-ชื่อเพจ/briefs/
  → ไฟล์ .txt ต้องมีเนื้อหาครบตามรูปแบบที่กำหนด (Storyboard, Image, Video prompts)
  → ถ้ามีส่วน THUMBNAIL PROMPT → บอทจะสร้างภาพ Thumbnail อัตโนมัติหลังซีนครบ
  → ถ้ามีส่วน Facebook Caption → ระบบจะไม่แตะส่วนนั้น เอาไว้ใช้เองสำหรับโพสต์

2.2 นำ Brief เข้าคิวในระบบ
  py bot.py queue

ผลลัพธ์: ระบบอ่าน Brief และสร้าง ID ให้โปรเจกต์ เช่น reel_0005
  จะเห็นข้อความ: "Queued: ชื่อไฟล์.txt -> reel_0005 (page-ชื่อเพจ)"

2.3 ตรวจสอบว่าเข้าคิวสำเร็จ
  py bot.py status

ถ้าเห็นโปรเจกต์ใหม่แสดงสถานะ "pending" แปลว่าพร้อมแล้ว

หมายเหตุ: ถ้าเปิด monitor.py ทิ้งไว้ ขั้นตอนที่ 2.2 จะทำโดยอัตโนมัติเมื่อวางไฟล์
  แต่ถ้าไม่ได้เปิด ต้องรัน py bot.py queue เอง


────────────────────────────────────────────────────────────────────────────────
ขั้นตอนที่ 3 — ดูสถานะโปรเจกต์
────────────────────────────────────────────────────────────────────────────────

ใช้เมื่อ: ต้องการดูว่าแต่ละโปรเจกต์อยู่ในขั้นตอนไหน

คำสั่ง:
  py bot.py status

ตัวอย่างผลลัพธ์:
  ID           PAGE             BRIEF                    STATUS                   IMG      VID
  ──────────────────────────────────────────────────────────────────────────────────────────
  reel_0005    page-Noble       pool ying.txt            pending                  0/10     0/10
  reel_0004    page-Noble       handiwork2.txt           images_done              10/10    0/10
  reel_0001    page-a           video1.txt               archived                 10/10    9/10

ความหมายของ STATUS:
  pending             → ยังไม่ได้เริ่มทำอะไร
  storyboard_done     → สร้าง Storyboard แล้ว กำลังรอสร้างภาพซีน
  images_done         → สร้างภาพทุกซีน + Thumbnail ครบแล้ว พร้อมสร้างวิดีโอ
  videos_in_progress  → สร้างวิดีโอบางซีนแล้ว ยังไม่ครบ
  videos_done         → สร้างวิดีโอครบทุกซีนแล้ว พร้อม archive
  archived            → archive แล้ว ไฟล์อยู่ใน ready/ พร้อมตัดต่อ CapCut
  not queued yet      → พบไฟล์ .txt แต่ยังไม่ได้รัน queue
  (brief renamed?)    → พบไฟล์ working/ อยู่ แต่หาไฟล์ .txt ต้นฉบับไม่เจอใน briefs/
                        แก้ไขโดย: ตรวจสอบว่าไฟล์ .txt ยังอยู่ใน pages/<เพจ>/briefs/ หรือไม่
                        ถ้าหายให้นำไฟล์ .txt กลับมาวางด้วยชื่อเดิมให้ตรงกัน


────────────────────────────────────────────────────────────────────────────────
ขั้นตอนที่ 4 — สร้างภาพ (Storyboard + ภาพซีน)
────────────────────────────────────────────────────────────────────────────────

ใช้เมื่อ: โปรเจกต์มีสถานะ pending และต้องการสร้างภาพ

คำสั่ง (ต้องระบุ ID เสมอ):
  py bot.py images reel_XXXX

ตัวอย่าง:
  py bot.py images reel_0005

สิ่งที่จะเกิดขึ้น:
  → Chrome เปิดขึ้นมาอัตโนมัติและไปที่ ChatGPT
  → ถ้ายังไม่ได้ login → ให้ login เอง บอทจะรอ 5 นาที
  → Phase A: บอทส่ง Storyboard Prompt → ChatGPT สร้างภาพ Storyboard 1 ภาพ
             → บอทดาวน์โหลดและบันทึกที่ working/
  → Phase B: บอทส่ง Image Prompt ทั้งหมด 10 ซีน → ChatGPT สร้างภาพทีละซีน
             → บอทดาวน์โหลดและบันทึกทีละภาพที่ working/
  → Phase C: บอทเปิด Chat ใหม่ใน ChatGPT → ส่ง Thumbnail Prompt + Character Sheet
             → ChatGPT สร้างภาพ Thumbnail 1 ภาพ → บอทบันทึกที่ working/
             (ข้ามขั้นตอนนี้อัตโนมัติถ้า Brief ไม่มีส่วน THUMBNAIL PROMPT)
  → เมื่อครบทุก Phase สถานะเปลี่ยนเป็น images_done

ระหว่างรอ: ห้ามปิด Terminal และ Chrome ที่บอทเปิดขึ้นมา
ใช้เวลา: ประมาณ 25-50 นาที ขึ้นอยู่กับความเร็ว ChatGPT (มี Phase C เพิ่มขึ้น)

หมายเหตุสำคัญ: ถ้า ChatGPT สร้างภาพผิดพลาดบางซีน บอทจะบันทึกเฉพาะซีนที่สำเร็จ
  สามารถรันคำสั่งเดิมซ้ำได้ บอทจะข้ามซีนที่มีอยู่แล้วและทำเฉพาะที่ขาด
  (Phase C ก็เช่นกัน — ถ้า thumbnail.png มีอยู่แล้ว Phase C จะถูกข้ามอัตโนมัติ)


────────────────────────────────────────────────────────────────────────────────
ขั้นตอนที่ 5 — สร้างวิดีโอ
────────────────────────────────────────────────────────────────────────────────

ใช้เมื่อ: โปรเจกต์มีสถานะ images_done และต้องการสร้างวิดีโอ

คำสั่ง (ต้องระบุ ID เสมอ):
  py bot.py videos reel_XXXX

ตัวอย่าง:
  py bot.py videos reel_0005

สิ่งที่จะเกิดขึ้น:
  → Chrome เปิดขึ้นมาอัตโนมัติและไปที่ Google Flow
  → ถ้ายังไม่ได้ login → ให้ login เอง แล้วรันคำสั่งใหม่
  → บอทอัปโหลดภาพซีนทีละซีน + ส่ง Video Prompt
  → Google Flow สร้างคลิปวิดีโอ → บอทดาวน์โหลดบันทึกที่ working/
  → ทำซ้ำจนครบทุกซีน → สถานะเปลี่ยนเป็น videos_done

ระหว่างรอ: ห้ามปิด Terminal และ Chrome ที่บอทเปิดขึ้นมา
ใช้เวลา: ประมาณ 30-60 นาที ขึ้นอยู่กับความเร็ว Google Flow

หมายเหตุสำคัญ:
  → ถ้าบางซีนสร้างไม่สำเร็จ สามารถรันคำสั่งเดิมซ้ำได้
    บอทจะข้ามซีนที่มีอยู่แล้วและทำเฉพาะที่ขาด
  → ถ้าบางซีนสร้างไม่ได้จริง ๆ (เช่น Google Flow มีข้อจำกัด)
    ก็รัน archive ได้เลย ไม่ต้องรอให้ครบ 10 ซีนก็ได้


────────────────────────────────────────────────────────────────────────────────
ขั้นตอนที่ 6 — Archive (เก็บไฟล์สำเร็จ)
────────────────────────────────────────────────────────────────────────────────

ใช้เมื่อ: สร้างวิดีโอเสร็จแล้ว (หรือต้องการปิดโปรเจกต์ แม้วิดีโอยังไม่ครบ)

คำสั่ง:
  py bot.py archive reel_XXXX

ตัวอย่าง:
  py bot.py archive reel_0005

สิ่งที่จะเกิดขึ้น:
  → ย้ายไฟล์ Brief .txt    จาก briefs/   → ready/reel_XXXX/
  → ย้าย storyboard.png    จาก working/  → ready/reel_XXXX/storyboard.png
  → ย้าย thumbnail.png     จาก working/  → ready/reel_XXXX/thumbnail.png  (ถ้ามี)
  → ย้ายวิดีโอทุกซีน       จาก working/  → ready/reel_XXXX/scene-01.mp4 ... scene-10.mp4
  → ลบไฟล์ภาพซีน .png ที่เหลือใน working/ ทิ้ง (ไม่จำเป็นแล้ว)
  → ลบโฟลเดอร์ working/ ถ้าว่างเปล่าแล้ว
  → อัปเดตสถานะใน contents.json เป็น "complete"

หลังจากนี้:
  → เปิดโฟลเดอร์ ready/reel_XXXX/ แล้วนำไปตัดต่อใน CapCut
  → เมื่อตัดต่อและอัปโหลดขึ้น Facebook เสร็จแล้ว
    สามารถลบโฟลเดอร์ ready/reel_XXXX/ ทิ้งได้เพื่อประหยัดพื้นที่
    (ระบบจะซ่อนโปรเจกต์นั้นออกจาก status อัตโนมัติ ไม่มี error)


────────────────────────────────────────────────────────────────────────────────
ขั้นตอนพิเศษ — รวบรวมโฟลเดอร์ที่ Archive แล้วไปไว้ในที่เดียว (collect)
────────────────────────────────────────────────────────────────────────────────

ใช้เมื่อ: Archive หลายโปรเจกต์แล้ว และต้องการรวบรวมทุกโฟลเดอร์ ready/ จากทุกเพจ
          มาไว้ในโฟลเดอร์ complete/ เพื่อให้หาไฟล์ตัดต่อได้สะดวกในที่เดียว

คำสั่ง:
  py bot.py collect

สิ่งที่จะเกิดขึ้น:
  → ค้นหาทุกโฟลเดอร์ pages/*/ready/reel_*/ ที่มีอยู่
  → ย้ายไปที่ complete/<ชื่อเพจ>/reel_XXXX/
  → แสดงจำนวนโฟลเดอร์ที่ย้าย

ตัวอย่างผลลัพธ์หลัง collect:
  complete/
  ├── 1-page-Strange-Frontiers/
  │   └── reel_0005/
  ├── 2-page-Reality-Survivor/
  │   └── reel_0007/
  └── 3-page-Noble-Handiwork/
      └── reel_0009/

หมายเหตุ: รันได้บ่อยเท่าที่ต้องการ ถ้าไม่มีโฟลเดอร์ ready/ ให้ย้าย จะแสดง "Nothing to collect"


────────────────────────────────────────────────────────────────────────────────
ขั้นตอนพิเศษ — เปลี่ยนชื่อโฟลเดอร์เพจ
────────────────────────────────────────────────────────────────────────────────

ใช้เมื่อ: ต้องการเปลี่ยนชื่อโฟลเดอร์เพจ เช่น เพิ่มตัวเลขนำหน้า หรือแก้ไขชื่อให้ถูกต้อง

คำสั่ง:
  py bot.py renamepage <ชื่อเก่า> <ชื่อใหม่>

ตัวอย่าง:
  py bot.py renamepage page-Noble-Handiwork 3-page-Noble-Handiwork

สิ่งที่จะเกิดขึ้น:
  → เปลี่ยนชื่อโฟลเดอร์ pages/page-Noble-Handiwork/ → pages/3-page-Noble-Handiwork/ บนดิสก์
  → อัปเดตชื่อเพจในทุกโปรเจกต์ใน contents.json ที่อ้างถึงชื่อเก่า

คำเตือน: ห้ามเปลี่ยนชื่อโฟลเดอร์ใน Windows Explorer โดยตรง
  ถ้าเปลี่ยนเองใน Explorer โดยไม่ใช้คำสั่งนี้ contents.json จะยังเก็บชื่อเก่าอยู่
  ทำให้ status แสดงข้อมูลผิดพลาด ต้องมาแก้ทีหลัง


────────────────────────────────────────────────────────────────────────────────
ขั้นตอนพิเศษ — อัปเดต Prompt หลังจาก Queue แล้ว
────────────────────────────────────────────────────────────────────────────────

ใช้เมื่อ: แก้ไข Prompt ในไฟล์ .txt หลังจากรัน queue ไปแล้ว แล้วต้องการให้ระบบใช้เวอร์ชันใหม่

สาเหตุที่ต้องใช้คำสั่งนี้:
  ระบบ copy prompt ทั้งหมดจากไฟล์ .txt เข้า contents.json ตอนรัน queue
  การแก้ไขไฟล์ .txt หลังจากนั้นไม่มีผล เพราะบอทอ่าน prompt จาก contents.json โดยตรง

คำสั่ง:
  py bot.py updateprompts reel_XXXX

ตัวอย่าง:
  py bot.py updateprompts reel_0008

สิ่งที่จะเกิดขึ้น:
  → ระบบอ่านไฟล์ .txt ใหม่และดึง Prompt ทุกซีน + Storyboard + Thumbnail มาอัปเดตใน contents.json
  → ไม่รีเซ็ต image_status, video_status หรือความคืบหน้าใด ๆ ทั้งสิ้น
  → บอทจะใช้ Prompt เวอร์ชันใหม่ในการรันครั้งถัดไป

หมายเหตุ: รันได้ทุกเมื่อ ไม่ว่าโปรเจกต์จะอยู่ในสถานะใดก็ตาม

  สรุปลำดับขั้นตอนทั้งหมดในการทำ 1 โปรเจกต์
================================================================================

  ครั้งแรกของเพจ (ทำเพียงครั้งเดียว):
  ┌─────────────────────────────────────────────────────────┐
  │  py bot.py addpage page-ชื่อเพจ                         │
  │  → วาง Character Sheet ลงใน briefs/                     │
  └─────────────────────────────────────────────────────────┘

  ทุกโปรเจกต์:
  ┌─────────────────────────────────────────────────────────┐
  │  1. วางไฟล์ Brief .txt ลงใน briefs/                     │
  │  2. py bot.py queue                                     │
  │  3. py bot.py status          (ตรวจว่าเห็น pending)     │
  │  4. py bot.py images reel_XXXX                          │
  │  5. py bot.py status          (ตรวจว่าเห็น images_done) │
  │  6. py bot.py videos reel_XXXX                          │
  │  7. py bot.py status          (ตรวจว่าเห็น videos_done) │
  │  8. py bot.py archive reel_XXXX                         │
  │  9. เปิด ready/reel_XXXX/ → ตัดต่อ CapCut → อัปโหลด   │
  └─────────────────────────────────────────────────────────┘


================================================================================
  คำถามที่พบบ่อย
================================================================================

Q: ลืมว่า ID ของโปรเจกต์คืออะไร?
A: รัน  py bot.py status  แล้วดูในคอลัมน์ ID

Q: บอทค้างหรือ Chrome ปิดไปเอง ทำอย่างไร?
A: ปิด Terminal แล้วเปิดใหม่ รันคำสั่งเดิมซ้ำได้เลย
   บอทจะข้ามงานที่ทำแล้วและทำต่อจากที่ค้างไว้

Q: ChatGPT หรือ Google Flow ให้ Login ใหม่?
A: Login ในหน้าต่าง Chrome ที่บอทเปิดขึ้นมา แล้วบอทจะทำงานต่อเองหรือรันคำสั่งใหม่

Q: ต้องการทำหลายเพจพร้อมกัน?
A: ได้ แต่ต้องเปิด Terminal คนละหน้าต่างและรันทีละโปรเจกต์
   Chrome ใช้ profile เดียวกัน ดังนั้นอย่ารัน images และ videos พร้อมกัน

Q: monitor.py คืออะไร ต้องเปิดไหม?
A: monitor.py เฝ้าดูโฟลเดอร์ briefs/ แล้วนำ Brief เข้าคิวอัตโนมัติเมื่อวางไฟล์
   ไม่จำเป็นต้องเปิด เพราะรัน  py bot.py queue  เองได้
   ถ้าเปิดทิ้งไว้ ให้เปิดใน Terminal แยกต่างหาก:  python monitor.py

Q: แก้ไข Prompt ในไฟล์ .txt แล้ว รันบอทใหม่ แต่ยังใช้ Prompt เก่าอยู่?
A: เพราะระบบ copy prompt เข้า contents.json ตอนรัน queue แล้ว การแก้ไขไฟล์ .txt
   ภายหลังไม่มีผล ให้รัน  py bot.py updateprompts reel_XXXX  ก่อน แล้วค่อยรันบอทใหม่

Q: ต้องการเปลี่ยนชื่อโฟลเดอร์เพจ ทำอย่างไร?
A: ห้ามเปลี่ยนชื่อใน Windows Explorer โดยตรง ให้ใช้คำสั่ง:
     py bot.py renamepage <ชื่อเก่า> <ชื่อใหม่>
   คำสั่งนี้เปลี่ยนชื่อโฟลเดอร์บนดิสก์และอัปเดต contents.json พร้อมกันในขั้นตอนเดียว

Q: ลบโฟลเดอร์ ready/reel_XXXX/ แล้วจะเกิดอะไร?
A: ไม่มี error โปรเจกต์นั้นจะหายออกจาก status อัตโนมัติ
   แต่ไฟล์จะหายถาวร ควรทำหลังตัดต่อและอัปโหลดเสร็จแล้วเท่านั้น


================================================================================
  หมายเหตุสำคัญ
================================================================================

  - ไม่ต้องแก้ไขไฟล์ contents.json เอง ระบบจัดการให้อัตโนมัติ
  - ห้ามแก้ไขหรือย้ายไฟล์ใน working/ ขณะบอทกำลังทำงาน
  - Character Sheet ต้องวางไว้ใน briefs/ ของเพจนั้นเสมอ อย่าลบออก
  - ชื่อโปรเจกต์ (reel_XXXX) ระบบกำหนดให้อัตโนมัติ ห้ามเปลี่ยนเอง
  - ถ้าย้ายโปรเจกต์ไปคอมพิวเตอร์เครื่องอื่น ให้คัดลอกทั้งโฟลเดอร์ไป
    แล้วติดตั้ง Python และ packages ใหม่ก่อนใช้งาน

================================================================================
  ★ อัปเดตใหม่ (2026): ระบบอัตโนมัติ + คำสั่งใหม่
================================================================================

--------------------------------------------------------------------------------
สรุปสิ่งที่เปลี่ยนไป (ของเดิม vs ของใหม่)
--------------------------------------------------------------------------------

ยังคงใช้ Extension สร้างงานเหมือนเดิมทุกอย่าง สิ่งที่เปลี่ยนคือ "คำสั่งที่เคยพิมพ์เอง
หลังสร้างวิดีโอเสร็จ" ตอนนี้ระบบทำให้อัตโนมัติแล้ว:

  ขั้นตอนหลังวิดีโอครบ          ของเดิม (พิมพ์เอง)              ของใหม่
  ─────────────────────────────────────────────────────────────────────────
  ย้ายไฟล์ไป ready/             py bot.py archive reel_XXXX     อัตโนมัติ
  รวมไป complete/              py bot.py collect               อัตโนมัติ
  ตัดต่อวิดีโอ                  เปิด Ai Auto Editor เอง          อัตโนมัติ
  เตรียมแคปชั่น                หา/เขียนเอง                     อัตโนมัติ (caption.txt)

→ ไม่ต้องพิมพ์ archive / collect / ตัดต่อ อีกต่อไป
→ รีลที่เสร็จจะไปอยู่ที่ complete/<เพจ>/<reel>/ พร้อมไฟล์ครบ: EDITED_*.mp4 + thumbnail.png + caption.txt
→ เหลือแค่ "โพสต์เอง" และ "เปลี่ยนชื่อโฟลเดอร์ลงท้าย .- หลังโพสต์"


--------------------------------------------------------------------------------
ขั้นตอนการทำงานแบบใหม่ (ใช้ Extension เป็นหลัก)
--------------------------------------------------------------------------------

  1. เปิด 2 Terminal แล้วรัน (เปิดทิ้งไว้ทั้งคู่ — เป็น "เครื่องยนต์" ที่ทำอัตโนมัติ):
        แท็บ 1:  python monitor.py        (ฝั่งสร้างงาน: บันทึกภาพ/วิดีโอ + archive + collect + caption)
        แท็บ 2:  python editor_queue.py   (ฝั่งตัดต่อ: ตัดต่อวิดีโอแต่ละรีล — แยกแท็บให้ log อ่านง่าย ไม่ปนกัน)
  2. เปิด Extension แล้วสร้างงานหลายรีลพร้อมกันเหมือนเดิม (3-5 รีล)
  3. ไม่ต้องทำอะไร — เมื่อวิดีโอครบทุกซีน ระบบจะ archive + collect + ตัดต่อ + เขียน caption.txt ให้เอง
  4. เปิดโฟลเดอร์ complete/<เพจ>/<reel>/ → โพสต์เองใน Meta Business Suite
     (อัปโหลด EDITED_*.mp4, ตั้ง thumbnail.png เป็นปก, วางข้อความจาก caption.txt)
  5. โพสต์เสร็จแล้ว เปลี่ยนชื่อโฟลเดอร์ลงท้ายด้วย ".-"  (= โพสต์แล้ว ระบบจะไม่แตะอีก)

  ** ทำไมต้องโพสต์เอง: การโพสต์ผ่าน API ทำให้ยอดวิวตก ต้องโพสต์เองแบบ Native ถึงได้ reach เต็ม **


--------------------------------------------------------------------------------
คำสั่งใหม่ทั้งหมด (ใช้เมื่อไร)
--------------------------------------------------------------------------------

  python monitor.py
      แท็บ 1 — เปิดทุกครั้ง เปิดทิ้งไว้ — ฝั่งสร้างงาน: บันทึกภาพ/วิดีโอ + archive + collect + caption

  python editor_queue.py
      แท็บ 2 — เปิดทุกครั้ง เปิดทิ้งไว้ — ฝั่งตัดต่อ: ตัดต่อวิดีโอแต่ละรีล
      (แยกออกจาก monitor.py เพื่อให้ log ไม่ปนกัน — รันพร้อม monitor.py ได้ปลอดภัย ไม่ตัดต่อซ้ำ)

  py bot.py force-complete reel_XXXX
      ปิดงานรีลที่ค้าง (เช่น ได้แค่ 9/10 ซีน) ด้วยคลิปเท่าที่มี → archive + เขียน caption ให้พร้อมโพสต์
      ** ไม่ตัดต่อเอง ** — editor_queue.py (แท็บ 2) จะตัดต่อให้ถัดไป
      ใช้เมื่อ Google Flow สร้างบางซีนไม่ได้จริง ๆ แล้วเรายอมรับเท่าที่มี — รันหลายรีลพร้อมกันได้

  py bot.py handoff [ชื่อเพจ]
      (สำรอง) เขียน caption.txt ให้รีลที่เสร็จแล้วทั้งหมด — ปกติไม่ต้องใช้เพราะระบบทำอัตโนมัติ
      ใช้กับรีลเก่าที่แคปชั่นว่าง: ตอนนี้ handoff ดึงแคปชั่นจากไฟล์ .txt ในโฟลเดอร์รีลให้เอง
      (ไม่ต้องรัน updateprompts ก่อนแล้ว)

  py bot.py pipeline <ชื่อเพจ>
      "เครื่องสำรอง" — ทำทั้งหมดเองโดยไม่ใช้ Extension (สร้างภาพ+วิดีโอ+ตัดต่อ+caption) ทีละรีล
      ใช้กับเครื่องสำรองที่ปล่อยให้ทำงานเอง  ** ห้ามรัน pipeline พร้อมกับ Extension **

  py bot.py cleanup
      ลบคลิปต้นฉบับ (scene-XX.mp4) ของรีลที่โพสต์แล้ว เพื่อคืนพื้นที่ดิสก์ (เก็บ EDITED ไว้)

  py bot.py preflight
      เคลียร์ Chrome ที่ค้าง + เช็คพื้นที่ดิสก์ ก่อนเริ่มรันรอบใหญ่

  py bot.py prune
      ย้ายรีลที่เสร็จ+เก็บแล้วออกจาก contents.json ไปไฟล์เก็บถาวร (ทำให้ไฟล์เล็ก เร็วขึ้น)
      เป็น dry-run ก่อน, สั่งจริงด้วย:  py bot.py prune apply

  notify.py (ไม่ต้องรันเอง)
      ระบบส่งแจ้งเตือนเข้า Telegram อัตโนมัติ ถ้าตั้งค่าไฟล์ notify_config.json
      (เช่น รีลเสร็จ, มี error, เจอ Cloudflare "Verify you are human")


--------------------------------------------------------------------------------
กฎสำคัญ: โฟลเดอร์ที่ลงท้าย ".-" = โพสต์แล้ว ห้ามแตะ
--------------------------------------------------------------------------------

  - โฟลเดอร์ที่เราเปลี่ยนชื่อให้ลงท้ายด้วย ".-" หมายถึง "โพสต์ขึ้นเพจแล้ว"
  - ระบบจะข้ามโฟลเดอร์เหล่านี้ทั้งหมด ไม่เขียน caption.txt ไม่ลบ ไม่แตะใด ๆ
  - handoff/cleanup/auto จะไม่ยุ่งกับโฟลเดอร์ ".-" เด็ดขาด


--------------------------------------------------------------------------------
แนวทางทำหลายเพจให้ได้ผลดี (Farm)
--------------------------------------------------------------------------------

  - สร้างเก็บไว้ล่วงหน้า (buffer) เช่น 1 สัปดาห์ → ถ้าวันไหน ChatGPT/Flow มีปัญหา ก็ยังมีของโพสต์
  - โพสต์กระจายเวลา อย่าโพสต์หลายเพจพร้อมกันทีเดียว
  - ทำคอนเทนต์ให้หลากหลายระหว่างเพจ (เทมเพลตเหมือนกันหมดทุกเพจเสี่ยงโดนลด reach)
